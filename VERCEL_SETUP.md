# Настройка переменных окружения в Vercel

## Как установить MONGODB_URI в Vercel

### Способ 1: Через веб-интерфейс Vercel (рекомендуется)

1. **Откройте ваш проект в Vercel:**
   - Перейдите на https://vercel.com/dashboard
   - Найдите и откройте проект **MoonCoon**

2. **Перейдите в настройки:**
   - В верхнем меню проекта нажмите на вкладку **Settings**

3. **Откройте Environment Variables:**
   - В левом боковом меню найдите раздел **Environment Variables**
   - Или перейдите напрямую: `Settings → Environment Variables`

4. **Добавьте переменную:**
   - Нажмите кнопку **Add New**
   - В поле **Name** введите: `MONGODB_URI`
   - В поле **Value** вставьте вашу строку подключения MongoDB
   - Выберите окружения:
     - ✅ **Production** (для продакшена)
     - ✅ **Preview** (для preview деплоев)
     - ✅ **Development** (опционально, для локальной разработки)
   - Нажмите **Save**

5. **Передеплойте проект:**
   - Перейдите во вкладку **Deployments**
   - Найдите последний деплой
   - Нажмите на три точки (⋯) рядом с деплоем
   - Выберите **Redeploy**
   - Или просто сделайте новый push в GitHub - Vercel автоматически задеплоит

### Способ 2: Через Vercel CLI

Если у вас установлен Vercel CLI:

```bash
# Установите Vercel CLI (если еще не установлен)
npm i -g vercel

# Войдите в аккаунт
vercel login

# Добавьте переменную окружения
vercel env add MONGODB_URI production
# Введите значение когда попросит

# Или для всех окружений сразу
vercel env add MONGODB_URI
```

### Формат строки подключения MongoDB

Ваша строка подключения должна выглядеть так:

**MongoDB Atlas (рекомендуется):**
```
mongodb+srv://username:password@cluster-name.mongodb.net/database-name?retryWrites=true&w=majority
```

**MongoDB локальный или другой хостинг:**
```
mongodb://username:password@host:port/database-name
```

### Где взять строку подключения?

1. **MongoDB Atlas:**
   - Войдите в https://cloud.mongodb.com
   - Выберите ваш кластер
   - Нажмите **Connect**
   - Выберите **Connect your application**
   - Скопируйте connection string
   - Замените `<password>` на ваш пароль
   - Замените `<dbname>` на имя вашей базы данных

2. **Другой MongoDB хостинг:**
   - Используйте строку подключения, предоставленную вашим провайдером

### Проверка

После добавления переменной и передеплоя:

1. Проверьте логи деплоя в Vercel - должно быть сообщение:
   ```
   ✅ MongoDB connected successfully
   ```

2. Если видите ошибку:
   ```
   ⚠️ MONGODB_URI not found
   ```
   - Убедитесь, что переменная добавлена для правильного окружения
   - Передеплойте проект

### Важные замечания

- ⚠️ **Никогда не коммитьте** `.env` файлы в Git
- ✅ Используйте разные базы данных для Production и Development
- ✅ Регулярно обновляйте пароли
- ✅ Используйте MongoDB Atlas для продакшена (бесплатный tier доступен)
